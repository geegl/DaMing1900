# 结构化规划层（design）

`OUTLINE.md` 继续作为剧情与节拍的长文本真源。  
`design/` 则作为章节生产流程的结构化控制层，专门保存不适合散落在大纲正文中的执行元数据。

## 目录职责

- `chapter_types.json`
  `41-100` 章的结构化章节类型真源。后续 `build_context_pack.py` 会从这里读取 `chapter_type`，不再依赖人工临场判断。

## 设计原则

1. `OUTLINE.md` 负责讲故事、讲节奏、讲长线推进。
2. `design/` 负责流程控制、执行元数据、强约束配置。
3. 结构化字段优先放在 `design/`，不要继续埋在不同 outline 段落里。
4. 后续如果增加：
   - 章节风险等级
   - 本章指定模型
   - 强制出场人物
   - 钩子类型
   都优先收敛到 `design/`。

## 当前强约束

- `41-100` 每章都必须先有 `chapter_type`
- `chapter_type` 只能是：
  - `normal`
  - `key`
- `key` 章用于重大节点、重大揭示、重大关系变化或幕结构拐点
