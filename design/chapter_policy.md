# 第1-280章章节类型判定规则

> 适用范围：`第1章 - 第280章`
>
> 本文件负责定义第一幕与第二幕内的章节类型判定标准，用于约束写作模型选择、质检强度和流程严谨度。

## 一、用途与使用方法

本文件不直接决定某一章的最终 `chapter_type`，它负责提供**判定规则**。  
真正的结构化章型真源始终是：

- `/Users/roven/Documents/Trae/Daming1900/design/chapter_types.json`

使用顺序必须固定为：

1. 先读：
   - `/Users/roven/Documents/Trae/Daming1900/BIBLE.md`
   - `/Users/roven/Documents/Trae/Daming1900/STYLE.md`
   - `/Users/roven/Documents/Trae/Daming1900/OUTLINE.md`
2. 再读：
   - `/Users/roven/Documents/Trae/Daming1900/design/chapter_policy.md`
3. 然后查看：
   - `/Users/roven/Documents/Trae/Daming1900/design/chapter_types.json`
4. 如果发现某章的 `chapter_type` 与本文件规则明显冲突：
   - 不得临时口头改章型
   - 应先修改 `chapter_types.json`
   - 再继续执行写作/审校流程

执行时的硬规则：

1. `chapter_types.json` 没有该章条目，停止执行。
2. 有条目但章型和本文件规则明显冲突，先修正规划，再执行。
3. 关键章不得降级为普通章。
4. 不得让模型自行决定章型。

## 二、核心原则

1. `OUTLINE.md` 决定剧情内容与节拍推进。
2. `design/chapter_types.json` 决定每章的结构化类型。
3. 本文件决定“什么样的章必须标成 `key`”。
4. 一旦 `chapter_type` 已写入 `design/chapter_types.json`，执行时不得临时改口。
5. 普通章和关键章的差异，不只是模型差异，更是结构强度、情绪重量、揭示级别与验收强度的差异。

## 三、章节类型定义

### `normal`

普通章承担以下一种或多种功能：

- 铺垫新冲突
- 推进调查线、技术线、宫廷线、海军线的小节点
- 发酵人物关系与情绪
- 为后续大章准备证据、人物、器物、补给节点或局势
- 提供阶段性爽点，但不改变整幕结构

普通章不等于“水章”。  
普通章仍然必须有：

- 具体场景开头
- 单章内的小破局
- 明确结尾钩子
- 足够支撑 `3500-4500` 字的事件密度

### `key`

关键章是会改变读者对本幕、主线、人物关系或战争格局认知的章节。  
只要满足以下任一条，就必须标为 `key`：

1. 幕结构拐点章
2. 主角身份跃迁章
3. 皇帝 / 郑玄机 / 刘恩 / 陆知微 / 顾惟新 / 沈青鸾 等核心关系发生性质变化的章节
4. 重大设定揭示章
5. 技术体系、政治结构、全球补给链或长线阴谋正式并轨的章节
6. 该幕的大爽点、大钩子、大托命、大悟道、大政变章节

## 四、第一幕（1-100）强制 `key` 判定标准

第一幕的关键章主要围绕：

- 谢长庚被看见、被纳入、被倚重、被托命
- 朱载机与谢长庚关系升级
- 郑玄机与刘恩阴谋线显影
- 兰芳 / 石油 / 帝国命脉线显影
- 格物总录、主机、主机立威、三案并轨
- 第一幕高潮：先帝病重、托命、驾崩、负录出京

以下类型在第一幕一律按 `key` 处理：

### 1. 幕结构节点

- 第一幕中段转折
- 第一幕倒计时启动
- 第一幕高潮前夜
- 第一幕高潮
- 第一幕尾声与第二幕承接点

### 2. 谢长庚身份跃迁

当谢长庚从以下任何一种状态跨到下一层时，该章必须是 `key`：

- 被看见
- 被纳入核心系统
- 从“会修机器的人”变成“能定是非的人”
- 从技术骨干变成托命对象

### 3. 朱载机与谢长庚关系升级

以下任何变化出现时，该章必须是 `key`：

- 由欣赏变成试探
- 由试探变成倚重
- 由倚重变成传承

### 4. 郑玄机与刘恩的阴谋线升级

以下任何变化出现时，该章必须是 `key`：

- 三案中的任一案由传闻进入明确证据层
- 苏州线、电报线、内廷线首次正式并轨
- 读者对“谁在布局”有了新层级认知

### 5. 格物工学重大突破

以下任何一种，必须是 `key`：

- 新舰提速
- 主机立威
- 总录线、油机线、主机线正式并轨
- 机械成功本身会改变朝局或主角地位

### 6. 兰芳 / 石油 / 帝国命脉线显影

当“兰芳不是背景，而是帝国命门”这一层被强化时，该章必须是 `key`。

## 八、怎么在实际流程里用

对于任意 `1-280` 章，执行时必须按以下顺序：

1. 从 `OUTLINE.md` 读取该章节拍
2. 从 `chapter_policy.md` 判断该章理论上应是 `normal` 还是 `key`
3. 从 `chapter_types.json` 读取该章实际结构化章型
4. 对比 2 和 3：
   - 一致：继续执行
   - 不一致：先修正规划，再执行
5. 按章型决定默认写作模型和终检模型
6. 进入正式 pipeline

如果某章满足本文件里的 `key` 条件，但在 `chapter_types.json` 里被标成 `normal`：

- 该章不得直接开写
- 不得临时口头降级
- 必须先修正 `chapter_types.json`

## 九、禁止事项

1. 不得因为赶进度把应为 `key` 的章节降成 `normal`
2. 不得在章节执行时临时口头改章型
3. 不得让模型自由决定章型
4. 不得跳过关键章更严格的写作与终检配置
5. 不得把“信息很多”误当成“关键章”
6. 不得把“场面很大”误当成“关键章”

判断关键章的标准永远是：

- 是否改变整幕结构
- 是否改变核心关系
- 是否改变读者认知
- 是否改变主角在世界中的位置
