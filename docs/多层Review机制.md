# 多层Review机制

## 一、Review层级设计

| 层级 | 频率 | 审查范围 | 审查重点 |
|------|------|---------|---------|
| **章节Review** | 每章 | 单章内容 | 世界观、POV、文风、番茄标准 |
| **批次Review** | 每10章 | 批次内10章 | 连贯性、伏笔、节奏、人物弧线 |
| **中期Review** | 每50章 | 前50章整体 | 结构完整性、主线推进、读者留存预测 |
| **阶段Review** | 每部（约70章）| 整部作品 | 主题深度、商业潜力、续作衔接 |

---

## 二、章节Review（每章）

**触发**：每章写完后

**审查内容**：
- 世界观一致性（禁用词、年号、技术代差）
- 物理逻辑（伤痕、时间线、装备）
- POV死锁（单视角、无上帝视角）
- 深层逻辑（动机、经济计算）
- 文风质量（AI痕迹词、感官反差）
- 番茄商业标准（开篇冲突、情绪曲线、金手指）

**输出**：
- 评分（Codex自定）
- 问题清单（P0-P3）
- 反向拷问
- 改进建议

---

## 三、批次Review（每10章）

**触发**：每10章完成后

**审查内容**：
1. **连贯性检查**
   - 人物伤痕是否一致（第1章烫伤 → 第10章是否还在）
   - 时间线是否正确（天工纪年、季节变化）
   - 伏笔是否回收或推进

2. **节奏分析**
   - 情绪曲线是否持续上行
   - 爽点分布是否合理（每章至少1个）
   - 压抑期是否过长（不超过3章）

3. **人物弧线**
   - 主角成长是否合理
   - 配角作用是否发挥
   - 反派威胁是否递进

4. **商业预测**
   - 完读率预估
   - 追读率预估
   - 弃书风险点

**输出**：
- 批次工作报告（Markdown）
- 问题汇总表
- 下批次改进建议
- Git链接

---

## 四、中期Review（每50章）

**触发**：每50章完成后

**审查内容**：
1. **结构完整性**
   - 三幕结构是否清晰
   - 主线是否推进
   - 支线是否合理

2. **读者留存预测**
   - 黄金三章效果
   - 第10章留存点
   - 第50章高潮点

3. **商业评估**
   - 是否符合番茄签约标准
   - 首秀预测
   - 完读率预测

**输出**：
- 中期评估报告（详细Markdown）
- 调整建议
- 后续50章规划

---

## 五、阶段Review（每部）

**触发**：每部完成后（第70/165/220章）

**审查内容**：
1. **主题深度**
   - 核心主题是否贯穿
   - 社会隐喻是否有效
   - 人物命运是否合理

2. **商业潜力**
   - 是否符合平台调性
   - 是否有续作价值
   - 是否有IP改编潜力

3. **续作衔接**
   - 伏笔是否预留
   - 悬念是否设置
   - 人物命运是否开放

**输出**：
- 阶段评估报告
- 续作规划
- 出版/影视改编建议

## 六、自动化实现

### 批次Review触发
```python
if chapter % 10 == 0:
    trigger_batch_review(batch_number=chapter // 10)
```

### 中期Review触发
```python
if chapter % 50 == 0:
    trigger_midterm_review(section=chapter // 50)
```

### 阶段Review触发
```python
if chapter in [70, 165, 220]:
    trigger_stage_review(part=1 if chapter <= 70 else 2 if chapter <= 165 else 3)
```
