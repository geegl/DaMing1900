## 🔄 在新机器上恢复工作环境

### 步骤1: 克隆仓库
```bash
git clone https://github.com/your-username/DaMing.git
cd DaMing
```

### 步骤2: 检查关键文件
```bash
# 检查记忆系统
ls memory/

# 检查Claude设置
ls .claude/

# 检查自动化状态
ls automation/state.json automation/foreshadowing_ledger.json

# 检查项目指令
ls CLAUDE.md
```

### 步骤3: 验证文件完整性
```bash
# 验证state.json结构
python3 -c "import json; json.load(open('automation/state.json'))"

# 验证foreshadowing_ledger.json结构
python3 -c "import json; json.load(open('automation/foreshadowing_ledger.json'))"
```

### 步骤4: 开始工作
```bash
# 查看当前进度
python3 automation/scripts/pipeline.py --status

# 生成下一章
python3 automation/scripts/pipeline.py --chapter 1
```

## ⚠️ 注意事项

### 1. settings.local.json 需要强制添加
```bash
git add -f .claude/settings.local.json
```

### 2. 不要忽略的文件
以下文件必须同步，不能加入 `.gitignore`：
- `memory/` 目录下的所有 `.md` 文件
- `automation/state.json`
- `automation/foreshadowing_ledger.json`

### 3. 可以忽略的文件
以下文件不需要同步（会自动生成）：
- `__pycache__/`
- `*.pyc`
- `*.log`
- `chapters/` 目录（生成的章节正文）

---

## 🚨 紧急恢复方案

如果不小心丢失了 `state.json` 或 `foreshadowing_ledger.json`：

### 方案1: 从最近的检查点恢复
```bash
python3 automation/scripts/checkpoint_manager.py --load 10
```

### 方案2: 从git历史恢复
```bash
git log --oneline automation/state.json
git checkout [commit-hash] -- automation/state.json
```

在任意机器上，运行以下命令确认环境正确：
