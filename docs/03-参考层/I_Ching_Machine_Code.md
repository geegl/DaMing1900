## 二、编码规则

### 卦象→二进制→机器指令

| 卦象 | 二进制 | 分析机指令 | 功能 | 使用频率 |
|------|--------|-----------|------|---------|
| **乾 ☰** | 111 | ADD（加法）| 炮弹弹道计算 | 高 |
| **坤 ☷** | 000 | STORE（存储）| 数据暂存 | 高 |
| **震 ☳** | 001 | JUMP（跳转）| 条件分支 | 中 |
| **巽 ☴** | 110 | MULTIPLY（乘法）| 复杂计算 | 中 |
| **坎 ☵** | 010 | COMPARE（比较）| 逻辑判断 | 中 |
| **离 ☲** | 101 | OUTPUT（输出）| 结果打印 | 低 |
| **艮 ☶** | 100 | HALT（停止）| 程序结束 | 低 |
| **兑 ☱** | 011 | INPUT（输入）| 读取纸带 | 低 |

## 三、纸带物理设计

### 每行6孔（对应一个卦象的6爻）

```
孔位：  1   2   3   4   5   6
        │   │   │   │   │   │
爻位：  初   二   三   四   五   上

阴爻（0）= 不打孔
阳爻（1）= 打孔
```

```
孔位：  1   2   3   4   5   6
        ●   ●   ●   ●   ●   ●
        │   │   │   │   │   │
爻象：  ━   ━   ━   ━   ━   ━  （六爻皆阳）
```

        │   │   │   │   │   │

## 四、指令集详细设计

### 1. ADD（加法）— 乾卦 ☰

**二进制**：111
**功能**：将两个寄存器的值相加

**格式**：
```
[乾卦] [寄存器A] [寄存器B] → [结果寄存器]

纸带示例：
孔1-6：●●●●●●（乾卦）
孔7-12：寄存器A编号
孔13-18：寄存器B编号
```

**应用场景**：
- 炮弹弹道计算
- 燃煤消耗统计
- 铁币流通统计

---

### 2. STORE（存储）— 坤卦 ☷

**二进制**：000
**功能**：将数据存入存储器

**格式**：
```
[坤卦] [数据] [地址] → 存储到指定地址

纸带示例：
孔1-6：○○○○○○（坤卦）
孔7-18：数据（二进制）

```

**应用场景**：
- 暂存中间计算结果
- 存储传感器数据
- 缓存频繁使用的数据

---

## 六、叙事应用

### 林霜降的逻辑病毒

**设计**：
- 表面是一段占卜程序（预测战争胜负）
- 实际是逻辑炸弹（让龙骨分析机死循环）

**代码片段**：
```
行1：  ○●●●●●（兑卦）●●●●●●（读取输入）
行2：  ○○●●●●（震卦）○○○○○○●●○○○（跳转到行2）
行3：  ●○○○●●（艮卦）（停止）
```

**效果**：
- 程序在第2行无限循环
- 机器卡死，无法执行防御计算
- 龙骨系统瘫痪

**叙事场景**：
- 林霜降将这段程序混在正常预测程序中
- 张廷远看不懂卦象，以为是迷信
- 洪威帝看懂了，但太晚
