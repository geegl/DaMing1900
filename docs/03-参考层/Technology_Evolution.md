## 二、天工差分机（第一部）

### 技术参数

| 项目 | 参数 |
|------|------|
| **结构** | 纯机械齿轮 |
| **功能** | 多项式计算（y=f(x)）|
| **精度** | 小数点后4位 |
| **速度** | 每分钟完成一次计算 |
| **输入** | 手动拨动齿轮 |
| **输出** | 齿轮位置读数 |
| **无逻辑分支** | 不能做条件判断 |

### 应用场景

| 场景 | 计算 |
|------|------|
| **炮弹弹道** | 根据初速度、发射角计算落点 |
| **燃煤消耗** | 根据航程、吨位计算耗煤量 |
| **铁币流通** | 统计各区铁币流通量 |

### 限制

**关键限制**：
- ❌ 无条件分支能力
- ❌ 不能读取纸带
- ❌ 不能存储程序
- ❌ 只能算固定公式

**叙事表现**：
- 陈铁修差分机，只是调整齿轮
- 林霜降无法用差分机编写逻辑病毒
- 张廷远用差分机做统计，但无法做复杂预测

## 三、洪威分析机（第二部）

### 技术参数

| 项目 | 参数 | 对比差分机 |
|------|------|-----------|
| **结构** | 机械齿轮+纸带读卡器 | 新增纸带机制 |
| **功能** | 条件分支+循环 | 新增逻辑判断 |
| **精度** | 小数点后6位 | 提升2位 |
| **速度** | 每秒完成一次运算 | 提升60倍 |
| **输入** | 打孔纸带 | 新增自动输入 |
| **输出** | 穿孔纸带+打印机 | 新增打印 |
| **逻辑分支** | 支持8种指令（卦象编码）| 新增 |

### 德国技术交易（1917年）

**大明获得**：
- 分析机核心图纸
- 纸带打孔机设计
- 逻辑门机械结构

**德国获得**：
- 大明舰队威慑英国
- 东非加煤站使用权
- 兰芳橡胶优先购买权

### 应用场景

| 场景 | 计算 | 差分机做不到 |
|------|------|------------|
| **复杂弹道** | 根据风速、温度修正 | 差分机只能算固定公式 |
| **战争模拟** | 模拟不同战术的结果 | 差分机无条件分支 |
| **后勤优化** | 最优化燃煤补给路线 | 差分机无循环能力 |
| **经济预测** | 预测铁币贬值速度 | 差分机无逻辑判断 |

### 卦象编码系统

**设计理念**：
- 表面：占卜符号（迷信）
- 实际：机器指令（技术）
- 详见《I_Ching_Machine_Code.md》

## 四、龙骨分析机（第三部）

### 技术参数

| 项目 | 参数 |
|------|------|
| **结构** | 分析机+蒸汽动力+液压系统 |
| **功能** | 防御系统控制+差分机火控 |
| **精度** | 小数点后8位 |
| **速度** | 每秒10次运算 |
| **输入** | 纸带+传感器（压力、温度）|
| **输出** | 蒸汽阀门控制+炮台旋转 |

### 集成系统

```
龙骨蒸汽防御系统
│
├── 分析机模块（卦象编码）
│   ├── 目标锁定计算
│   ├── 火控解算
│   └── 弹道预测
│
├── 蒸汽动力模块
│   ├── 高压蒸汽管道
│   ├── 液压驱动炮台
│   └── 冷却系统
│
└── 传感器网络
    ├── 压力传感器
    ├── 温度传感器
    └── 位置传感器
```

### 林霜降的逻辑病毒

**原理**：
- 利用分析机的逻辑分支能力
- 注入死循环程序
- 导致分析机卡死

**代码**：
```
行1：INPUT（读取传感器数据）
行2：JUMP 行2（跳转到行2）
行3：HALT（永远执行不到）
```

**效果**：
- 分析机无限循环
- 无法计算目标锁定
- 炮台无法旋转
- 防御系统瘫痪

## 五、技术细节对比

### 差分机 vs 分析机

| 维度 | 天工差分机 | 洪威分析机 |
|------|-----------|-----------|
| **计算方式** | 多项式逼近 | 逻辑运算 |
| **输入方式** | 手动拨齿轮 | 纸带自动输入 |
| **程序存储** | 无 | 纸带存储 |
| **条件分支** | 无 | 8种指令 |
| **循环能力** | 无 | 支持 |
| **计算速度** | 1次/分钟 | 1次/秒 |
| **精度** | 4位小数 | 6位小数 |
| **应用场景** | 固定公式计算 | 复杂逻辑运算 |
