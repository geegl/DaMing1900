## 🛡️ 八层防护体系 v4.0（强制执行）

### 1. 物理层面（防失忆）

**规则**：
- ❌ 死人不可复活
- ❌ 残疾不可恢复
- ❌ 伤痕不可消失
- ❌ 物品特征不可遗忘

**检查方式**：
- 每章生成前读取 `automation/character_physical_profiles.json`
- 每章生成后运行 `automation/scripts/state_verification.py`

**示例**：
```json
"陈铁": {
  "永久伤痕": {
    "左腿旧伤": {
      "来源": "第3章阿牛之死时被蒸汽烫伤",
      "位置": "左小腿外侧",
      "叙事功能": "提醒陈铁阿牛的死"
    }
  }
}
```

---

### 2. 语言层面（防同质化）

**规则**：
- 每人必须使用 `character_voice_profiles.json` 中的独特语言特征
- 平均句长标准差 ≥ 4.5
- 禁用工具人台词："心头一颤"、"双目赤红"、"总而言之"

**检查方式**：
- 自动扫描禁用台词
- 句长分布统计

**示例**：
| 人物 | 语言特征 | 典型台词 |
|------|---------|---------|
| 陈铁 | 短促、直接、动作导向 | "修好了。"、"明天开工。" |
| 老鬼 | 残缺、市井、方言 | "他娘的，又炸了。"、"狗日的洋人……" |
| 朱靖镇 | 极短、命令式、金属压迫 | "造。"、"朕知道了。" |

---

### 3. 叙事层面（防脱钩）

**规则**：
- 每章生成前读取 `foreshadowing_ledger.json`
- 必须至少回收1个旧伏笔 + 埋1个新伏笔
- 主线/支线/感情线必须有明确交叉

**检查方式**：
- 伏笔回收验证
- 咬合张力检测

---

### 4. 文体层面（防漂移）

**规则**：
- ❌ 禁用词：微微、淡淡、隐约、仿佛、似乎、由此可见、从叙事角度分析
- ❌ 禁用腔调：分析腔、论文腔、总结腔、客服腔

**检查方式**：
- 自动扫描禁用词库
- AI味词汇检测

---

### 5. 质量层面（防注水+疲劳）

**规则**：
- 核心冲突推进检查：每章必须有明确冲突推进 ≥ 1处
- 动作描写占比 ≥ 25%
- 张力密度对比前章不得下降 > 10%
- 细节密度：每1000字 ≥ 12处具体感官细节

**检查方式**：
- 冲突推进度统计
- 张力密度对比
- 细节密度追踪

---

### 6. 情感层面（防缺失）

**规则**：
- 每章必须包含至少1处角色内心挣扎 + 人性深度描写
- 关键章节强制高潮验证

**检查方式**：
- 情感弧光追踪
- 高潮质量验证

---

### 7. 视觉层面（防降噪）v4.0 ⭐

**规则**：
- 每章强制 ≥ 3处强感官反差（脏/乱/血腥/煤油味 vs 精致/庄严）
- 中性平滑词上限 ≤ 3次/章
- 必须使用 `visual_noise_library.json` 中的晚清蒸汽噪声

**示例**：
```
✅ 正确：煤烟落在米饭上，牙齿咬到煤渣，发出脆响
❌ 错误：微微的煤灰落在饭上，似乎有点脏
```

---

### 8. 视角层面（防POV死锁）v4.0 ⭐

**规则**：
- 每章开头必须声明 POV：【POV: 角色名 - 第X视角】
- 严格禁止上帝视角、无标注跳视角
- 参考 `pov_lock_system.json`

**示例**：
```
【POV: 陈铁 - 第1视角】
```

## 🎨 去AI味三板斧（强制注入）

### 1. 参考文本注入

**每章生成前自动注入**：
- `docs/03-参考层/Reference_Text_Library.md`
- 3段参考文本：公文体/说书味/宫廷反差

**来源**：
- 《陶庵梦忆》：晚明市井生活
- 《金瓶梅》：明代对话、底层真实感
- 《南明史》：南明历史细节

---

### 2. 毛边噪声注入

**允许 5-8% 毛边噪声**：
- 断句、重复、时代错位俚语、方言
- 破格修辞、口语化表达

**示例**：
```
✅ 正确："他娘的，又炸了。"（老鬼的方言）
✅ 正确："造。"、"朕知道了。"（朱靖镇的极短句）
❌ 错误："他感到非常愤怒和绝望。"（AI平滑腔）
```

---

### 3. 温度动态控制

**规则**：
- 前30章：温度 0.85（允许更多创造性）
- 后续章节：温度 0.75 + 故意破格指令（更稳定但保留毛边）

## 🤖 子代理工作模式（v5.0核心升级）

### 工作流程

```
1. 规划代理（Planning Agent）
   ↓ 读取 Daming1900_Master_Outline.md
   ↓ 更新 state.json
   ↓ 生成 Beat Sheet（本章核心冲突+情感弧光）

2. 写作代理（Writing Agent）
   ↓ 读取 Beat Sheet
   ↓ 读取前2-3章摘要 + 人物档案
   ↓ 注入参考文本（Reference_Text_Library.md）
   ↓ 八层防护自检
   ↓ 生成章节草稿

3. 质控代理（Quality Agent）
   ↓ 运行 state_verification.py（物理状态检查）
   ↓ 运行 death_pit_guard.py（六大死穴检测）
   ↓ 运行 visual_pov_guard.py（视觉+视角检测）
   ↓ 运行 quality_checker.py（质量综合检查）
   ↓ 更新 state.json

4. 后处理（每10章）
   ↓ AI味润色 pass（只改文风不动剧情）
   ↓ 生成进度报告
   ↓ 提交到GitHub
```
