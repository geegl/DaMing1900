## 一、时间线质控原则

### 核心原则

**时间线是小说的生命线，一致性是重中之重。**

1. ✅ **严格按时间顺序推进**
   - 第1章：1900年
   - 第70章：1909年
   - 不允许时间跳跃、倒流、穿越

2. ✅ **每章必须标注时间**
   - 格式：`1900年十月十五`
   - 置于章节开头

3. ✅ **时间连续性验证**
   - 每章写完后，检查与前章的时间间隔
   - 合理间隔：1天至3个月
   - 异常间隔：>6个月需特别说明

## 三、时间线质控Agent

### Agent职责

**时间线审计员（Timeline Auditor）**

**触发时机**：每章写完后立即运行

**检查项**：

1. **提取章节时间**
   ```python
   def extract_chapter_time(chapter_content):
       """提取章节开头的时间标记"""
       pattern = r'^(\d{4})年(\d{1,2})月(\d{1,2})日?'
       match = re.search(pattern, chapter_content)
       return match.groups() if match else None
   ```

2. **验证时间连续性**
   ```python
   def validate_timeline_continuity(prev_chapter_time, current_chapter_time):
       """验证时间连续性"""
       prev_date = datetime(*prev_chapter_time)
       curr_date = datetime(*current_chapter_time)

       # 检查时间是否倒流
       if curr_date < prev_date:
           return {"error": "时间倒流", "prev": prev_date, "curr": curr_date}

       # 检查时间间隔是否合理
       delta = curr_date - prev_date
       if delta.days > 180:  # 超过6个月
           return {"warning": "时间间隔过大", "days": delta.days}

       return {"valid": True}
   ```

3. **验证时间线范围**
   ```python
   def validate_timeline_range(chapter_num, year):
       """验证章节号是否匹配时间线范围"""
       expected_range = get_expected_year_range(chapter_num)
       if not expected_range[0] <= year <= expected_range[1]:
           return {"error": "年份超出预期范围", "expected": expected_range, "actual": year}
       return {"valid": True}
   ```

## 五、时间线错误处理

### 错误级别

| 级别 | 描述 | 处理方式 |
|------|------|---------|
| **P0（致命）** | 时间倒流、时间穿越 | 立即停止，重写章节 |
| **P1（严重）** | 时间跳跃>6个月 | 补充过渡剧情或分章 |
| **P2（一般）** | 时间间隔不合理 | 调整时间标记 |
