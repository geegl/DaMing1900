## 二、核心工具

### 1. Codex审查系统

**文件**：`automation/prompts/codex_review_template.txt`

**功能**：
- 世界观一致性检查（清朝年号、现代元素）
- 物理逻辑检查（伤痕、时间线、装备）
- POV死锁检查（单视角、无上帝视角）
- 深层逻辑检查（动机、经济计算）
- 文风质量检查（AI痕迹词汇）

**用法**：
```bash
# 写完章节后运行
codex review --uncommitted < automation/prompts/codex_review_template.txt
```

**输出格式**：
- 总分：XX/50
- P0问题（致命）：世界观错误、物理逻辑错误
- P1问题（严重）：POV违规、深层逻辑错误
- P2问题（一般）：文风问题
- P3建议：改进建议

**通过标准**：
- 总分 ≥ 40/50
- 无P0问题
- P1问题 ≤ 2个

---

### 2. 世界观验证脚本

**文件**：`automation/scripts/worldview_validator.py`

**功能**：
- 检查禁用词（清朝年号、清朝官职、现代元素）
- 检查年号计算正确性

**用法**：
```bash
python3 automation/scripts/worldview_validator.py chapters/第XXX章.md
```

**输出**：
- ✅ 通过世界观验证
- ❌ 发现致命错误 + 错误列表

---

### 3. 批次管理系统

**文件**：`automation/scripts/batch_manager.py`

**功能**：
- 创建批次日志（每10章一批）
- 记录Codex审查结果
- 记录纠错日志
- Git提交
- Telegram通知

**批次日志位置**：`progress/batches/batch_XX_log.md`

---

### 4. Telegram通知

**功能**：
- 批次完成通知

**用法**（MCP工具）：
```python

    title="标题",

    priority="normal"  # low, normal, urgent

```

## 三、工作流程（每章）

### 第1步：写章节

使用Claude Code的8-agent系统写章节：
- 规划Agent（Haiku）
- 世界观约束Agent（Haiku）
- 写作Agent（Sonnet）
- 逻辑检查Agent（Sonnet）
- 情感高潮Agent（Sonnet）
- 风格润色Agent（Sonnet）
- 质控Agent（Sonnet）
- **Codex审查Agent（Opus）** ← 新增

### 第2步：Codex审查

```bash
# 查看章节内容
cat chapters/第XXX章_标题.md

# 运行Codex审查
codex review --uncommitted < automation/prompts/codex_review_template.txt
```

### 第3步：根据审查结果修正

**P0问题**：必须修复
**P1问题**：建议修复
**P2问题**：可选修复

修正后重新运行Codex审查，直到通过。

### 第4步：世界观验证

```bash
python3 automation/scripts/worldview_validator.py chapters/第XXX章.md
```

必须通过验证才能提交。

### 第5步：提交Git

```bash
git add chapters/第XXX章_*.md
git commit -m "完成第XXX章：标题

- 字数：XXXX字
- Codex得分：XX/50
- P0问题：X个
- P1问题：X个

Co-Authored-By: Claude Haiku 4.5 <noreply@anthropic.com>"

git push
```

### 第6步：更新批次日志

编辑 `progress/batches/batch_XX_log.md`：
- 更新章节状态表
- 添加Codex审查记录
- 添加纠错日志（如有）

---

## 四、批次流程（每10章）

### 批次结束时的任务

1. **统计批次数据**：
   - 总字数
   - 平均Codex得分
   - P0/P1问题总数

2. **更新批次日志**：
   - 填写"四、批次总结"
   - 记录经验教训

3. **提交批次到Git**：
   ```bash
   git add progress/batches/batch_XX_log.md
   git commit -m "完成Batch XX：第XXX-XXX章"
   git push
   ```

4. **发送Telegram通知**：
   ```python
   mcp__codepilot-notify__codepilot_notify(
       title=f"📦 《大明1900》Batch XX 完成",
       body=f"""
   📊 数据统计：
   - 章节：第XXX章 - 第XXX章
   - 总字数：XXXX字
   - 平均Codex得分：XX/50
   - P0问题：X个
   - P1问题：X个

   ✅ 已同步到GitHub
       """,
       priority="normal"
   )
   ```
